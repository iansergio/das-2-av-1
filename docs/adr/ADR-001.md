# ADR-001: Estratégia de Ingestão de Dados com Azure Functions

**Status:** Aceito | **Data:** 2026-03-13 | **Autores:** Equipe de Desenvolvimento

## Contexto

A VendaMais Distribuidora Ltda. utiliza um ERP proprietário que centraliza os dados dos módulos de vendas, estoque, financeiro e logística. Apesar de atender às operações do dia a dia, o sistema não possui mecanismos automatizados de extração de dados para análise, dependendo de exportações manuais em planilhas.

O projeto tem como objetivo automatizar a extração diária dos dados transacionais do ERP, garantindo que essas informações possam ser utilizadas na construção de dashboards com defasagem máxima de 24 horas.

Entre as restrições definidas:

- Não realizar modificações no ERP existente
- Automatizar o processo de extração
- Garantir rastreabilidade dos dados
- Utilizar a plataforma Microsoft Azure

Na arquitetura proposta, a camada de ingestão é responsável por extrair os dados do ERP e disponibilizá-los para as próximas etapas do pipeline.

## Decisão

Foi definida a utilização de Azure Functions com execução agendada para realizar a ingestão dos dados do ERP.

A solução consiste em:

Uma função serverless executada por agendamento (timer)
Conexão com a API ou banco de dados do ERP
Extração dos registros do dia
Armazenamento dos dados no Azure Blob Storage

Essa abordagem implementa a camada de ingestão conforme a arquitetura definida, garantindo a automatização do processo de extração de dados.

## Consequências

(+) Automatização da extração de dados do ERP

(+) Redução da dependência de processos manuais

(+) Integração com a arquitetura baseada em Azure

(+) Disponibilização dos dados para as próximas etapas do pipeline

(-) Dependência da disponibilidade do ERP para execução da extração

(-) Processamento realizado de forma agendada (não em tempo real)

## Alternativas rejeitadas

- Extração manual de dados: rejeitada devido à necessidade de automação do processo

- Modificação direta no ERP: rejeitada pois está fora do escopo do projeto

- Outras formas de ingestão não baseadas em Azure Functions: rejeitadas por não estarem alinhadas à arquitetura definida na solução
