# ADR-002: Estratégia de Armazenamento com Azure SQL Database

**Status:** Aceito | **Data:** 2026-03-13 | **Autores:** Equipe de Desenvolvimento

## Contexto

Após a etapa de ingestão dos dados do ERP, é necessário definir onde esses dados serão armazenados para que possam ser utilizados nas análises da plataforma de Inteligência Operacional.

Como o objetivo do projeto é gerar dashboards e indicadores para apoio à tomada de decisão, os dados precisam estar organizados e estruturados para facilitar consultas e relatórios.

Também é importante que a solução escolhida esteja integrada com o ambiente da Microsoft Azure utilizado na arquitetura do projeto e que permita crescimento conforme a quantidade de dados aumenta ao longo do tempo.


## Decisão

Foi definido o uso do Azure SQL Database como banco de dados principal para armazenar os dados analíticos da plataforma.

Os dados extraídos do ERP são armazenados inicialmente no Azure Blob Storage em formato bruto. 

Depois de passarem por etapas de tratamento e organização, eles são carregados em tabelas relacionais no Azure SQL Database.

Esse banco passa a funcionar como a principal fonte de dados para os dashboards desenvolvidos no Power BI.


## Consequências

(+) Estrutura organizada para consultas e análises

(+) Integração direta com Power BI

(+) Escalabilidade e alta disponibilidade na nuvem

(-) Dependência do ecossistema Microsoft Azure

(-) Custos contínuos do serviço em nuvem


## Alternativas rejeitadas

- Banco de dados local: exigiria infraestrutura própria e manutenção

- Uso apenas de arquivos no Azure Blob Storage: dificultaria consultas analíticas

- Bancos NoSQL: menos adequados para modelo relacional usado em BI
